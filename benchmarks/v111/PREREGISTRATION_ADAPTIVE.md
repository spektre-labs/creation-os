# v111.2 — Pre-registration: task-adaptive σ router

**Locked on:** 2026-04-20 (UTC) — this document is the pre-registration
of the adaptive σ-matrix for `creation-os-kernel` at the HEAD commit
recorded in `benchmarks/v111/PREREGISTRATION_ADAPTIVE.lock.json`
(generated by `preregister_adaptive.py` on the same day).  Everything
below was fixed **before** any split-sample evaluation was run on this
branch.

## 0 — Why

The first v111.2 adaptive matrix (previous commit) was declared
post-hoc: the composite / task-routed signals in
`adaptive_signals.py` were designed **after** the pre-registered v111
matrix was analysed, and therefore reported under Bonferroni N = 12
with an explicit "post-hoc exploration, not pre-registered" banner.

This document promotes exactly those signals to pre-registered status
on a data split the post-hoc analysis never saw.

## 1 — Hypothesis (exact)

> **H₁ (router):**  On a held-out test split of the existing v103
> σ-sidecars, `sigma_task_adaptive` achieves a lower AURCC than the
> `entropy` baseline in at least one of the four pre-registered task
> families, at family-wise α = 0.05 under Bonferroni N = 12.

Null hypothesis H₀: `sigma_task_adaptive` does not improve AURCC on
any family at the family-wise α.

## 2 — Signals under test

The three adaptive signals are exactly those exported by
`benchmarks/v111/adaptive_signals.py` as of the lock commit; a SHA-256
of that file is written into
`PREREGISTRATION_ADAPTIVE.lock.json::signals_sha256` and is verified
at analysis time.  No signal may be modified without a new
pre-registration (a new lock file + new split seed).

| # | signal | definition |
|---|---|---|
| 1 | `sigma_composite_max`  | `max(σ_max_token, entropy)` — conservative OR-abstain |
| 2 | `sigma_composite_mean` | `0.5·(σ_max_token + entropy)` — linear blend |
| 3 | `sigma_task_adaptive`  | per-task pre-specified routing (see §3) |

`sigma_task_adaptive` routing (frozen):

```
truthfulqa_mc2 → sigma_max_token   (v111 pre-registered winner)
hellaswag      → entropy           (v111 matrix-strong baseline)
arc_challenge  → sigma_product     (v104 winner)
arc_easy       → entropy           (v111 matrix-near-tied)
else           → sigma_product     (v104 default)
```

## 3 — Split protocol

- Seed: `0xC05A1A2A` (constant, committed in the lock file).
- Per task family, shuffle document ids deterministically, then cut at
  the halfway point:
    - **Calibration split (50 %)** — used to compute a conformal
      quantile τ (see §4); NEVER used to score any AURCC claim.
    - **Test split (50 %)** — used ONLY to score the pre-registered
      hypothesis.  Never touched before the lock file exists.
- Document ids in each split are persisted into the lock file so that
  any re-analysis is bit-identical to the first.

## 4 — Conformal τ (v111.2-conformal, reported alongside)

For every (task, signal) pair we also compute a per-task conformal
threshold on the calibration split using the Vovk–Gammerman
quantile:

    n   = |calibration set|
    q   = ⌈(1 − α) (n + 1)⌉           (α = 0.05)
    τ_c = (q-th smallest σ among documents the model got CORRECT)

This gives the **finite-sample coverage guarantee**: on exchangeable
draws from the same distribution, the subset `σ ≤ τ_c` retains
accuracy ≥ 1 − α in expectation.  The guarantee is exchangeability-
based and honest about distribution shift (see
`docs/v111/CONFORMAL_GUARANTEE.md`).

## 5 — Statistical test

- Metric: AURCC (area under the risk-coverage curve; lower is better).
- Test: paired bootstrap against the entropy baseline, `n_boot = 2000`,
  RNG seed fixed (`seed=0`).
- Multiple-comparison correction: Bonferroni N = 12 (3 signals × 4
  task families), α_fw = 0.05 / 12 = 0.00417.
- Reject H₀ iff `delta_mean < 0 AND p_two_sided < α_fw` on the TEST
  split for at least one (task, signal) pair in the registry.

## 6 — What counts as success vs failure

- **Success:** at least one (task, signal) pair passes the Bonferroni
  criterion on the TEST split.  Report every such pair in
  `frontier_matrix_adaptive.md` under a heading
  `Pre-registered test-split results (lock: <sha>, seed: 0xC05A1A2A)`.
- **Failure:** no pair passes.  Report the matrix honestly, label all
  rows `not replicated on test split`, and DO NOT re-promote the
  post-hoc claim.  A failed pre-registration is a real result.

## 8 — Invariants enforced by the check script

`benchmarks/v111/check_v111_prereg_adaptive.sh` refuses to pass if:
- the lock file is missing, or its `signals_sha256` disagrees with the
  current `adaptive_signals.py`,
- the split halves drift from the lock file,
- the Bonferroni N is not 12,
- the report fails to label the test-split results as
  `preregistered=true`.
